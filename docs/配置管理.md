# 配置管理规范

SolarMcp 采用 **「Core 管理 Core，Plugin 管理 Plugin」** 的配置模式。

## 原则

| 范围 | 配置文件 | 内容 |
|------|----------|------|
| Core（框架） | `config.yaml` | server、logging、thread_pool、auth、`plugins.directory`、内置 `tools.read_file` |
| Plugin | `plugins/{name}/*.yaml` | 各插件业务参数，Core **不解析** |

## Core 配置示例

```yaml
server:
  host: "0.0.0.0"
  port: 8090

plugins:
  directory: "./plugins/"    # 扫描子目录，非扁平 *.so
  autoload: true
  allow_reload: true

tools:
  read_file:                 # 仅此段为内置工具配置
    enabled: true
    max_size_mb: 10
    allowed_paths: ["/tmp", "/home"]

auth:
  enabled: true
  token: "..."
```

**不要**在 `config.yaml` 中放置 `tools.shell` 等插件业务字段。

## 插件目录布局

```
plugins/
├── shell/
│   ├── shell_plugin.so
│   └── shell.yaml
└── example/
    ├── minimal_plugin.so
    └── minimal.yaml
```

### 配置文件发现顺序

对每个 `plugins/{subdir}/*.so`，Core 在同目录按序查找：

1. `plugin.yaml`
2. `{subdir}.yaml` — 如 `shell/shell.yaml`
3. `{so_basename}.yaml` — 如 `shell_plugin.yaml`

找到则通过 ABI `get_plugin_config_path()` 传给插件；未找到则传空字符串（插件使用默认值）。

## 插件侧

插件在 `mcp_plugin_register()` 中调用 `get_plugin_config_path()`，自行解析 YAML/JSON：

```yaml
# plugins/shell/shell.yaml
enabled: true
timeout_sec: 30
max_output_mb: 10
allowed_shells: ["/bin/sh", "/bin/bash"]
```

## 生命周期

```
读取 config.yaml → 得到 plugins.directory
        ↓
扫描 plugins/*/ 子目录
        ↓
dlopen(*.so) + 发现 *.yaml
        ↓
mcp_plugin_init(host_api)   // host_api 含 get_plugin_config_path
        ↓
mcp_plugin_register()       // 插件读自己的 yaml
        ↓
卸载: unregisterTool → mcp_plugin_shutdown → dlclose
```

## 热重载

`plugins/reload` 会重新扫描子目录并重新读取各插件 yaml，**无需**修改 `config.yaml`。
