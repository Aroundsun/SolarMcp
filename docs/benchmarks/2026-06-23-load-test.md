## 简要结论

1. **低并发**（10 连接）下 `tools/list` 约 **416 QPS**，p99 延迟 < 32ms。
2. **50 连接**时 QPS 反而降至 **296**，p50 升至 **145ms**——单 Reactor 排队效应明显。
3. `read_file` / `plugin_demo` 在 20 连接下 QPS 更高，因每连接请求数较少、总压测时长更短；不宜与 A/B 直接横比绝对 QPS。
4. 若要更准确对比，建议 Release 构建 + 固定总请求数（如各场景均 5000 请求）后复测。
