# 插件热重载指南

本文说明如何在**不重启 `solarmcpd`** 的情况下，重新加载 `plugins/lib/` 目录下的 `.so` 动态库。

## 目录说明

插件相关文件分两层存放：

| 路径 | 内容 |
|------|------|
| `plugins/shell/`、`plugins/filesystem/` 等 | 插件**源码**与桥接代码 |
| `plugins/lib/*.so` | 编译后的**动态库**（运行时由服务器加载） |

`config.yaml` 中的 `plugins.directory` 指向 **`plugins/lib/`**，不要指到源码根目录。

```
plugins/
├── shell/              ← 源码
├── filesystem/         ← 源码
└── lib/                ← 运行时加载
    ├── shell_plugin.so
    └── read_file_plugin.so
```

更多说明见 [plugins/README.md](../plugins/README.md)。

## 背景

SolarMcp 通过 `dlopen` 在启动时扫描 `plugins/lib/`，加载其中的 `.so` 并向 `ToolManager` 注册 MCP 工具（如 `shell`）。开发插件时若每次改代码都重启服务器，迭代会很慢。

**热重载**流程：

1. 卸载已加载插件（注销其工具 → `dlclose`）
2. 重新扫描 `plugins.directory`
3. 再次 `dlopen` 并注册工具

内置工具（如 `read_file`）**不会**被卸载，仅插件注册的工具会受影响。

```
┌─────────────┐     plugins/reload      ┌──────────────────┐
│  MCP 客户端  │ ──────────────────────► │  PluginManager   │
└─────────────┘                         │  unloadAll()     │
                                        │  loadFromDir()   │
                                        └────────┬─────────┘
                                                 │
                    ┌────────────────────────────┼────────────────────────────┐
                    ▼                            ▼                            ▼
             注销插件工具                   dlopen(.so)                    tools/list
             （如 shell）              mcp_plugin_register                立即可见新工具
```

## 前置条件

| 条件 | 说明 |
|------|------|
| 插件已构建 | `cmake --build build` 会将 `.so` 复制到 `plugins/lib/` |
| 服务器已启动 | `./build/solarmcpd --config config.yaml` |
| 热重载已启用 | `config.yaml` 中 `plugins.allow_reload: true`（默认开启） |
| 认证（推荐） | `auth.enabled: true` 时，须先在同一 TCP 连接上完成 `authenticate` |

## 配置

`config.yaml` 相关项：

```yaml
plugins:
  directory: "./plugins/lib/"   # 动态库目录，扫描其中 *.so
  autoload: true                  # 启动时自动加载
  allow_reload: true              # 是否开放 plugins/reload 接口
```

- `allow_reload: false` — 禁用热重载接口，只能重启服务器加载新插件
- 修改 `tools.shell` 等配置后，热重载会重新读取 `config.yaml`（插件实现了 `mcp_plugin_set_config_path` 时）

## 操作步骤

### 1. 启动服务器

```bash
mkdir -p logs
./build/solarmcpd --config config.yaml
```

日志中应出现类似：

```
[INFO] Plugin loaded: shell_plugin (./plugins/lib/shell_plugin.so, 1 tool(s) registered)
[INFO] Loaded 1 plugin(s) from ./plugins/lib/
```

### 2. 修改并重新编译插件

例如修改 `plugins/shell/shell_tool.cpp` 后：

```bash
cmake --build build --target shell_plugin -j$(nproc)
```

构建完成后 `plugins/lib/shell_plugin.so` 会自动更新。

### 3. 触发热重载

**推荐：专用脚本**

```bash
python3 scripts/reload_plugins.py
python3 scripts/reload_plugins.py --list-tools   # 重载后列出工具名
python3 scripts/reload_plugins.py -q             # 仅输出摘要
```

**手动 JSON-RPC**

在同一 TCP 连接上依次发送（Content-Length 分帧）：

1. 认证（若 `auth.enabled: true`）

```json
{"jsonrpc":"2.0","method":"authenticate","params":{"token":"你的令牌"},"id":1}
```

2. 热重载

```json
{"jsonrpc":"2.0","method":"plugins/reload","params":{},"id":2}
```

3. 验证工具列表

```json
{"jsonrpc":"2.0","method":"tools/list","params":{},"id":3}
```

### 4. 查看响应

`plugins/reload` 成功时返回：

```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "unloaded": 1,
    "loaded": 1,
    "failed": 0,
    "tools": 2
  }
}
```

| 字段 | 含义 |
|------|------|
| `unloaded` | 本次卸载的插件数量 |
| `loaded` | 重新加载成功的插件数量 |
| `failed` | 扫描到但加载失败的 `.so` 数量 |
| `tools` | 热重载后工具总数（含内置工具） |

服务器日志（`logs/solarmcp.log`）示例：

```
[INFO] Reloading plugins from './plugins/lib/' (unloading 1 plugin(s))
[INFO] Plugin unloaded: shell_plugin (./plugins/lib/shell_plugin.so)
[INFO] Plugin loaded: shell_plugin (./plugins/lib/shell_plugin.so, 1 tool(s) registered)
[INFO] Plugin reload complete: unloaded=1, loaded=1, failed=0, tools=2
```

## C++ 编程接口

在自有代码中集成：

```cpp
mcp::PluginReloadResult result =
    plugin_manager.reloadFromDirectory(plugin_dir, tool_manager, config_path);

// 热重载单个插件
bool ok = plugin_manager.reloadPlugin(
    "./plugins/lib/shell_plugin.so", tool_manager, config_path);
```

`PluginManager::unloadAll(tool_manager)` 会注销插件工具并 `dlclose`，**不影响**内置 `read_file` 等工具。

## 生命周期与卸载顺序

热重载与进程退出时须遵守：

```
注销插件工具 (unregisterTool)  →  dlclose(.so)
```

若先 `dlclose` 再析构 `Tool`，会触发 ASan 崩溃。请使用 `unloadAll(tool_manager)` / `reloadFromDirectory()`，不要手动 `dlclose`。

`main.cpp` 中对象声明顺序：

```cpp
mcp::PluginManager plugin_manager;  // 先声明
mcp::ToolManager tool_manager;      // 后声明 → 析构时先销毁 Tool
```

## 常见问题

### 热重载后工具没变化？

- 确认 `plugins/lib/*.so` 修改时间已更新
- 查看 `logs/solarmcp.log` 是否有加载失败或重名警告
- 内置 `read_file` 与 `read_file_plugin.so` 重名时，插件版会注册失败（`failed` 可能大于 0）

### 返回「Method not found」？

- 检查 `plugins.allow_reload` 是否为 `true`
- 确认已用含该接口的版本重新编译并重启过 `solarmcpd`

### 返回「Authentication required」？

- 先在同一连接上调用 `authenticate`
- 或使用 `python3 scripts/reload_plugins.py`（会自动处理认证）

### 热重载时正在执行 tools/call？

当前实现**未对进行中的工具调用加锁**，重载瞬间若有请求在执行可能失败。建议在低流量时重载，或稍后重试。

### 生产环境建议

- 保持 `auth.enabled: true`，更换默认令牌
- 不需要运行时重载时，设置 `plugins.allow_reload: false`
- `shell` 等高危工具仅在受信网络暴露
